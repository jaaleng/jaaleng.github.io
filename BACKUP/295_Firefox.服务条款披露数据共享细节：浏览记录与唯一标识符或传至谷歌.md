# [Firefox 服务条款披露数据共享细节：浏览记录与唯一标识符或传至谷歌](https://github.com/jaaleng/jaaleng.github.io/issues/295)

**Firefox 服务条款披露数据共享细节：浏览记录与唯一标识符或传至谷歌**

最近看到不少人在讨论 Firefox 的服务条款和隐私政策更新，里面提到会把某些浏览数据、唯一标识符共享给合作伙伴，包括谷歌云平台（Google Cloud Platform）。这事儿一出，隐私圈里又热闹起来了。

我自己平时主要用 Firefox，图的就是它比 Chrome 更注重隐私一点。看到这个消息后，赶紧去翻了翻 Mozilla 官方的隐私声明。

![Firefox 隐私与安全设置页面](https://i.829259.xyz/api/rfile/Screenshot_20260329_235505_com_twitter_android_MainActivity.jpg)

简单说，Firefox 确实会处理一些浏览数据（比如访问网站的类别、顶级域名，甚至特定 URL）、唯一标识符（client_id、session_id、cookie 标识符等），以及技术数据、位置、语言偏好这些东西。这些数据在某些情况下会共享给服务供应商和合作伙伴，用于提供服务、分析、改进产品，或者营销相关目的。其中谷歌云就被列在里面，主要用于云端分析服务。

但要注意，并不是说你的完整浏览历史记录全都被打包发给谷歌。很多数据还是留在你本地设备上，不会自动发到 Mozilla 服务器，除非你开启了某些功能。

比如 Google 安全浏览服务（反钓鱼、恶意软件防护），浏览器会和谷歌服务器通信，上传部分哈希后的网址信息来检查风险。这功能很多浏览器都有，目的是保护你，但确实会涉及数据交换。

![Google Safe Browsing 拦截警告页面](https://i.829259.xyz/api/rfile/Screenshot_20260329_235522_com_twitter_android_MainActivity.jpg)

为什么会共享呢？Mozilla 解释是为了正常运行浏览器、提升安全、做数据分析，以及维持商业运营。之前他们 FAQ 里还明确说过“永不卖用户数据”，后来因为法律定义问题，就改得更中性了。

作为普通用户，我的感觉是：Firefox 比 Chrome 还是强不少（Chrome 登录账号后浏览记录直接进谷歌生态），但它也不是完全“零数据共享”的理想状态。尤其是用了云服务、AI 功能或者安全防护时，总会有点妥协。

![Firefox vs Chrome 隐私对比](https://i.829259.xyz/api/rfile/Screenshot_20260329_235449_com_twitter_android_MainActivity.jpg)

生活在西安的我，平时上网看新闻、捣鼓工具，挺在意这些细节的。谁都不想自己的浏览习惯被莫名其妙拿去分析吧。

那怎么办？有几点可以试试：

- 进 Firefox 设置 → 隐私与安全，关掉不必要的遥测和数据收集选项；
- 谨慎开启 Google 安全浏览，如果在意可以换其他防护方式；
- 用 uBlock Origin、Strict 模式增强跟踪保护；
- 或者考虑 Librewolf、Mullvad Browser 这些硬核 fork 版本，它们把很多 Mozilla 默认连接都砍掉了。

当然，完全不共享数据几乎不可能，除非你用纯本地工具。但至少知道条款里写了什么，就能自己决定要不要继续用。

这个时代，浏览器厂商都要平衡隐私和商业，Firefox 算是在中间摇摆的那个。看到条款里明确提到浏览数据和唯一标识符可能传给谷歌（或其他伙伴），我还是有点小失望，但也算提醒大家：隐私从来不是默认的，得自己动手管。

你还在用 Firefox 吗？看到这个更新后，有没有调整设置，或者换浏览器了？欢迎评论区说说你的看法。
